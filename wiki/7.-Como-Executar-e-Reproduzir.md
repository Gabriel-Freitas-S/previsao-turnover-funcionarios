# Como Executar e Reproduzir o Projeto

Esta página descreve o passo a passo necessário para preparar o ambiente local de desenvolvimento, executar o pipeline de treinamento de Machine Learning, gerar os slides em PDF e rodar o painel interativo web.

---

## 📋 Pré-requisitos
* **Python**: Versão 3.8 ou superior instalada no sistema.
* **Navegador Web**: Chrome, Firefox, Edge, Safari ou qualquer navegador moderno com suporte a WebAssembly.

---

## 🛠️ 1. Configuração do Ambiente Virtual Python
Antes de rodar qualquer script, recomenda-se isolar as dependências do projeto em um ambiente virtual (`virtualenv`):

```bash
# 1. Clone o repositório ou navegue até a pasta raiz do projeto
cd previsao-turnover-funcionarios

# 2. Crie o ambiente virtual com o nome '.venv'
python3 -m venv .venv

# 3. Ative o ambiente virtual
# No Linux ou macOS:
source .venv/bin/activate

# No Windows (PowerShell):
# .venv\Scripts\Activate.ps1
# No Windows (Prompt de Comando):
# .venv\Scripts\activate.bat
```

Após ativar o virtualenv, atualize o instalador de pacotes e instale as dependências:

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

As dependências principais incluem: `pandas`, `numpy`, `scikit-learn`, `matplotlib`, `seaborn`, `joblib` e `fpdf2`.

---

## 🤖 2. Executando o Pipeline de Machine Learning
Para realizar todo o processo de carregamento de dados, análise estatística preliminar, plotagem de gráficos EDA individuais e consolidados, particionamento dos dados, otimização de hiperparâmetros, treinamento de modelos de classificação e exportação do modelo campeão:


## 🌐 5. Executando a Interface Web Localmente
A interface web interativa é definida no arquivo [index.html](file:///home/gabriel-freitas-souza/Projetos/previsao-turnover-funcionarios/index.html). 

> [!WARNING]
> **Aviso de Restrição CORS:** Ao tentar abrir o arquivo `index.html` diretamente clicando duas vezes sobre ele (protocolo `file://`), o navegador bloqueará as chamadas AJAX (`fetch`) de download do modelo `modelo_turnover_pyodide.pkl` e do arquivo `slides/metrics.json` por motivos de segurança (Políticas de Origem Cruzada - CORS).
>
> **Solução**: Deve-se servir o arquivo através de um servidor HTTP local.

### Como rodar o servidor HTTP local:

#### Opção A (Recomendada via Terminal/Python):
Com o terminal aberto na pasta raiz do projeto, execute:
```bash
python -m http.server 8000
```
Em seguida, abra seu navegador e acesse:
```
http://localhost:8000
```

#### Opção B (No VS Code):
Instale a extensão **Live Server**, abra a pasta do projeto no VS Code, abra o arquivo `index.html` e clique no botão **Go Live** localizado na barra inferior do editor.
