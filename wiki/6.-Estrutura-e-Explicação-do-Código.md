## 📄 src/main.py
Este script contém o pipeline completo de treinamento, otimização e avaliação de modelos. Ele automatiza o processo desde a leitura dos dados até a exportação final dos arquivos de produção.

### Principais Funções:

*   **`load_data(path)`**:
    *   *O que faz*: Lê o arquivo CSV bruto do dataset utilizando `pandas`. Adiciona suporte ao encoding `utf-8-sig` para evitar bugs de leitura com acentos em português.
*   **`explore_data(df)`**:
    *   *O que faz*: Realiza a análise estatística preliminar no terminal. Exibe a dimensionalidade (`shape`), checa a presença de valores nulos (que no dataset é $0$) e imprime a proporção de registros em cada classe da variável `saiu`.
*   **`plot_eda(df)`**:
    *   *O que faz*: Gera de forma automatizada o painel geral de gráficos exploratórios contendo 6 subplots com distribuições de turnover em relação a satisfação, horas de trabalho, avaliações, departamento e salários. Salva o painel combinado em `slides/eda_plots.png` e exporta os 6 gráficos individualizados para compor os slides da apresentação.
*   **`preprocess_data(df)`**:
    *   *O que faz*: Realiza a limpeza de strings residuais nas colunas categóricas e separa a variável alvo `saiu` ($y$) da matriz de features preditoras ($X$).
*   **`build_preprocessor(X)`**:
    *   *O que faz*: Constrói os transformadores do Scikit-Learn. Define o `StandardScaler` para as variáveis numéricas e o `OneHotEncoder(drop='first', handle_unknown='ignore')` para as variáveis categóricas, agrupando-os no `ColumnTransformer` que gerencia o fluxo de transformação de forma unificada.
*   **`create_models()`**:
    *   *O que faz*: Inicializa as instâncias de classe de cada modelo (`LogisticRegression`, `RandomForestClassifier` e `GradientBoostingClassifier`). Define os dicionários contendo os hiperparâmetros de busca (grids) para cada classificador.
*   **`train_and_evaluate(...)`**:
    *   *O que faz*: Executa o loop principal de treinamento. Para cada modelo:
        1. Encapsula o preprocessador e o classificador em um `Pipeline`.
        2. Configura e roda o `GridSearchCV` acoplado a um `StratifiedKFold(5)` no treino.
        3. Realiza predições de probabilidade e classe nas bases de validação e teste.
        4. Coleta métricas de F1-Score, acurácia, precisão, recall, matriz de confusão e calcula importâncias de atributos.
*   **`plot_results(results, y_test)`**:
    *   *O que faz*: Gera e salva gráficos comparativos consolidados em `slides/model_comparison.png` (como gráficos de barras de métricas lado a lado, plot das curvas ROC, matrizes de confusão térmicas e o ranking dos 10 atributos de maior impacto preditivo).
*   **`main()`**:
    *   *O que faz*: Orquestrador principal. Executa as funções anteriores em sequência. Faz a divisão estratificada 60/20/20 dos dados de treino, validação e teste. Compara o F1-Score de validação de cada modelo treinado, seleciona o campeão (Gradient Boosting) e o persiste fisicamente no arquivo binário `modelo_turnover.pkl` usando a biblioteca `joblib`. Por fim, exporta as métricas de teste obtidas para `slides/metrics.json` e encerra gerando as visualizações gráficas.

## 📄 src/export_pyodide_model.py
Script utilitário encarregado de ler o modelo completo gerado pelo Scikit-Learn e extrair cirurgicamente apenas seus componentes matemáticos essenciais, descartando metadados de dependência de pacotes Python tradicionais para rodar no frontend web com tamanho reduzido.

### Lógica Operacional:
1. Carrega o arquivo binário `modelo_turnover.pkl`.
2. Acessa as propriedades internas do pipeline transformador (`preprocessor`).
3. Extrai as listas das médias e escalas (`mean_` e `scale_`) do `StandardScaler`.
4. Extrai a lista de arrays de categorias (`categories_`) e o índice do descarte (`drop_idx_`) do `OneHotEncoder`.
5. Acessa as propriedades internas de árvores do classificador `GradientBoostingClassifier` (`estimators_`).
6. Para cada estimador (árvore binária de decisão), extrai o mapeamento topológico em listas:
   * `children_left`: índices do nó filho esquerdo.
   * `children_right`: índices do nó filho direito.
   * `feature`: índice do atributo testado naquele nó de decisão.
   * `threshold`: limiar numérico de divisão do atributo.
   * `value`: vetor de predição folha.
7. Agrupa todas as listas extraídas em um dicionário simplificado e salva em `modelo_turnover_pyodide.pkl` utilizando o protocolo serializador 4 do Python.

## 📄 src/generate_slides.py
Script que gera um relatório profissional em formato PDF com 10 slides no padrão A4 Paisagem, eliminando a necessidade de design manual de slides.

### Principais Classes e Funções:

*   **`class SlidePDF(FPDF)`**:
    *   Subclasse de `FPDF` customizada para desenho de slides.
    *   **`header(self)`**: Sobrescreve o cabeçalho padrão para desenhar uma linha de estilo no topo direito e o título do projeto em todas as páginas após a capa.
    *   **`footer(self)`**: Sobrescreve o rodapé para inserir o número da página dinâmico "Slide X/10" precedido por uma linha azul escura.
    *   **`add_title_slide(title, subtitle)`**: Renderiza a capa do projeto com títulos de grandes dimensões centralizados e dados institucionais (nomes dos alunos e disciplina).
    *   **`add_section_slide(section_num, title)`**: Cria um slide de transição ou introdução de seção com barra divisória espessa.
    *   **`add_body_text(text, size)`**: Imprime um bloco corrido de texto formatado.
    *   **`add_bullet(text, size, bold_prefix)`**: Cria marcadores de lista recuados com suporte para prefixos em negrito.
    *   **`add_image_centered(img_path, w)`**: Insere uma imagem (como os gráficos EDA ou de comparação) centralizada horizontalmente na largura útil da página.
*   **`build_presentation()`**:
    *   *O que faz*: Abre o arquivo `slides/metrics.json` para capturar os dados reais de acurácia, precisão, recall, F1 e ROC-AUC do último treinamento executado.
    *   Monta sequencialmente as 10 páginas da apresentação:
        1. Capa.
        2. Introdução e justificativa de negócio.
        3. Obtenção e Pré-processamento dos dados.
        4. Resultados e imagens gráficos da análise exploratória (EDA).
        5. Eixo Secundário de Regressão (Bike Sharing e Data Leakage).
        6. Descrição teórica dos classificadores e GridSearchCV.
        7. Modelagem da Regressão e K-Fold.
        8. Métricas conceituais (F1-Score, RMSE, MAE, R²).
        9. Tabela comparativa real preenchida dinamicamente com dados do JSON e imagem das curvas ROC.
        10. Conclusão, Insights práticos e link do repositório.
    *   Exporta o documento final compilado para `slides/apresentacao_turnover.pdf`.

## 📄 index.html
Interface web completa escrita em HTML5 estrutural, CSS3 (com design premium baseado em variáveis de cor CSS, sombras fluidas, transições de hover e glassmorphism) e JavaScript para controle do estado de visualização e inferência WASM.

### Lógica da Inferência no Frontend (JavaScript + Pyodide):
*   **Inicialização**:
    ```javascript
    async function initPyodide() {
      // 1. Carrega o interpretador WASM do Pyodide
      window.pyodide = await loadPyodide();
      // 2. Importa o módulo pickle e numpy na memória WASM
      await window.pyodide.loadPackage(["numpy"]);
      // 3. Carrega o arquivo modelo_turnover_pyodide.pkl via Fetch API
      let response = await fetch("modelo_turnover_pyodide.pkl");
      let buffer = await response.arrayBuffer();
      // 4. Passa o buffer binário para o Python dentro do Pyodide decodificar
      window.modelData = pyodide.runPython(...);
    }
    ```
*   **Predição Matemática**:
    Quando o formulário é enviado, o JavaScript coleta os valores ajustados nos inputs e sliders e executa em Python (dentro do Pyodide):
    1.  **Padronização das Numéricas**:
        Para cada feature numérica $i$:
        $$x_scaled_i = \frac{x_i - mean_i}{scale_i}$$
    2.  **One-Hot Encoding**:
        Identifica as categorias textuais de salário e departamento informadas, criando as flags $0$ ou $1$ equivalentes e descartando a primeira coluna.
    3.  **Avaliação sequencial dos estimadores (Gradient Boosting)**:
        *   Inicia a estimativa com o log-odds a priori do modelo (`init_raw`).
        *   Para cada uma das 100 árvores de decisão presentes no modelo exportado:
            *   Percorre a árvore de cima para baixo. Em cada nó interno, avalia se o valor da feature de entrada é menor ou igual ao `threshold` daquele nó.
            *   Desce para o filho esquerdo (`children_left`) se verdadeiro, ou para o filho direito (`children_right`) se falso.
            *   Ao atingir um nó folha (identificado quando o filho esquerdo é $-1$), extrai o valor numérico predito pela árvore.
            *   Soma esse valor ao acumulador de predição geral, multiplicado pelo fator `learning_rate` ($0,1$).
        *   Converte o acumulador final de log-odds em probabilidade de turnover ($p$) aplicando a função sigmoide:
            $$p = \frac{1}{1 + e^{-\text{acumulador}}}$$
    4.  Exibe o resultado na tela com barras animadas, agrupa o colaborador em uma faixa de risco (baixo, moderado ou alto) e sugere ações de retenção personalizadas para o RH de acordo com os principais pontos de insatisfação identificados no formulário.
