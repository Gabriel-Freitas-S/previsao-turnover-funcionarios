# Modelos de Machine Learning e Otimização

Para solucionar o problema de classificação binária de turnover, foram selecionados três algoritmos com filosofias arquiteturais distintas. Essa diversidade permite comparar desde modelos lineares altamente interpretáveis até métodos baseados em comitês (ensembles) de árvores não-lineares de alta capacidade preditiva.

---

## 🤖 Algoritmos Selecionados

### 1. Regressão Logística
* **Como funciona**: Modelo linear que calcula uma combinação ponderada dos atributos de entrada e mapeia o resultado real para uma probabilidade entre $0$ e $1$ através da função matemática logística (Sigmoide):
  $$P(Y=1|X) = \sigma(\beta_0 + \beta_1 X_1 + \dots + \beta_p X_p) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 X_1 + \dots + \beta_p X_p)}}$$
* **Finalidade**: Serve como linha de base (baseline) interpretável. A importância de cada variável é mapeada diretamente pela magnitude e sinal dos coeficientes ajustados $\beta_j$.
* **Tratamento de Desbalanceamento**: Configurado com `class_weight='balanced'`.

### 2. Random Forest Classifier
* **Como funciona**: Comitê de árvores de decisão baseado em aprendizado por agregação (Bagging - *Bootstrap Aggregating*). Treina múltiplas árvores independentes em subamostras aleatórias dos dados e das features. A predição final da floresta é definida pela votação majoritária de todas as árvores.
* **Finalidade**: Modelo robusto que lida de forma nativa com relações não-lineares, não exige suposições de normalidade nos dados e é altamente resistente a overfitting por conta da aleatorização na seleção de atributos.
* **Tratamento de Desbalanceamento**: Configurado com `class_weight='balanced'`.

### 3. Gradient Boosting Classifier
* **Como funciona**: Comitê de árvores de decisão baseado no paradigma de Boosting. Diferente do Random Forest (onde as árvores são paralelas e independentes), as árvores no Gradient Boosting são geradas de forma sequencial. Cada nova árvore é ajustada para prever e corrigir os erros (resíduos) cometidos pelo conjunto de árvores anterior, descendo na direção do gradiente da função de perda.
* **Finalidade**: Obter o maior limite de acerto possível. Costuma apresentar o estado da arte do desempenho preditivo para dados estruturados/tabulares.

---

## ⚖️ Estratégia class_weight='balanced'
Dado o desbalanceamento das classes (~76% permaneceu / ~24% saiu), um classificador convencional focaria excessivamente em acertar a classe majoritária, uma vez que isso garante uma acurácia básica de 76% sem esforço.

Para mitigar isso, aplicamos a estratégia de **Pesos de Classe Balanceados** (`class_weight='balanced'`) na Regressão Logística e no Random Forest. Esse algoritmo ajusta a penalização de erros na função de perda de forma inversamente proporcional à frequência das classes:
$$w_j = \frac{N_{total}}{N_{classes} \times N_j}$$
Onde:
* $w_j$ é o peso atribuído à classe $j$.
* $N_{total}$ é o total de amostras (8.999 no treino).
* $N_{classes}$ é o número de classes (2).
* $N_j$ é a contagem de amostras da classe $j$.

Dessa forma, errar um funcionário que de fato saiu voluntariamente (`saiu` = 1) custa aproximadamente **3 vezes mais caro** para a otimização matemática do que errar um funcionário que ficou, forçando o classificador a aprender os padrões da classe minoritária.

## ⚙️ Ajuste de Hiperparâmetros via GridSearchCV
Para extrair a máxima capacidade de generalização de cada classificador, foi realizada uma varredura exaustiva de hiperparâmetros usando **Busca em Grade (GridSearchCV)** acoplada com **Validação Cruzada Estratificada de 5 Folds** (`StratifiedKFold(n_splits=5)`) aplicada estritamente sobre a base de treino.

A validação cruzada divide o conjunto de treino em 5 partes, treinando o modelo em 4 delas e validando na parte restante de forma rotativa. Isso garante estabilidade nas estimativas de desempenho.

### Espaços de Busca Definidos no GridSearchCV

```python
param_grids = {
    "RegressaoLogistica": {
        "classifier__C": [0.01, 0.1, 1, 10],            # Força inversa de regularização L2
        "classifier__penalty": ["l2"],
        "classifier__solver": ["lbfgs"],
    },
    "RandomForest": {
        "classifier__n_estimators": [100, 200],          # Número de árvores na floresta
        "classifier__max_depth": [5, 10, None],          # Profundidade máxima das árvores
        "classifier__min_samples_split": [2, 5],        # Amostras mínimas para dividir um nó interno
        "classifier__min_samples_leaf": [1, 2],          # Amostras mínimas necessárias em um nó folha
    },
    "GradientBoosting": {
        "classifier__n_estimators": [50, 100],           # Número de etapas de boosting (árvores)
        "classifier__learning_rate": [0.05, 0.1, 0.2],   # Taxa de encolhimento de cada árvore
        "classifier__max_depth": [3, 5],                 # Profundidade máxima das árvores individuais
    },
}
```

### Métrica de Otimização: F1-Score
A busca em grade foi configurada para selecionar a melhor combinação de parâmetros otimizando o **F1-Score** (parâmetro `scoring='f1'`). Isso garante que os hiperparâmetros escolhidos maximizem simultaneamente a precisão dos alertas gerados e a cobertura (recall) de identificação de turnover de colaboradores.
