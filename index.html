1 Definição do Problema

Problema: Prever se um funcionário tem alta probabilidade de deixar a empresa (Turnover / Attrition). Trata-se de um problema de Classificação Binária:

  • Classe Positiva (1): Funcionário saiu da empresa (Yes)
  • Classe Negativa (0): Funcionário permaneceu (No)

Relevância: A saída repentina de funcionários que dominam fluxos documentais críticos gera custos de recrutamento, perda de conhecimento institucional e riscos à conformidade legal. A retenção preditiva de talentos é uma prioridade estratégica para RH.

2 Obtenção e Preparação dos Dados

Dataset: IBM HR Analytics Employee Attrition & Performance

Fonte: Kaggle

  • 1470 registros, 35 atributos (26 numéricos, 9 categóricos)
  • 0 valores nulos — dados previamente higienizados
  • Variável alvo: Attrition (Yes/No)

Pré-processamento

  • Remoção de colunas sem variância: EmployeeCount, StandardHours, Over18
  • Remoção do identificador: EmployeeNumber
  • Codificação One-Hot para 7 variáveis categóricas
  • StandardScaler para variáveis numéricas
  • ColumnTransformer + Pipeline para evitar data leakage
  • Divisão: 80% treino / 20% teste (com stratified sampling)

4 Metodologia

Modelos Selecionados

  • Regressão Logística (class_weight='balanced') — Modelo linear interpretável
  • Random Forest (class_weight='balanced') — Ensemble não-linear robusto

Otimização

  • GridSearchCV com validação cruzada StratifiedKFold (5 folds)
  • Métrica de otimização: F1-Score (classe minoritária)
  • Estratégia de balanceamento: class_weight='balanced'

Grid — Regressão Logística

  • C (regularização inversa): [0.01, 0.1, 1, 10, 100]
  • penalty: L2 (Ridge)
  • solver: lbfgs

Grid — Random Forest

  • n_estimators: [100, 200, 300]
  • max_depth: [5, 10, None]
  • min_samples_split: [2, 5]
  • min_samples_leaf: [1, 2]

6 Discussão dos Resultados

🎯 F1-Score

A Regressão Logística obteve melhor F1-Score (0,48) e Recall (0,68), demonstrando maior capacidade de identificar funcionários que realmente vão sair da empresa.

📊 Acurácia Enganosa

O Random Forest obteve maior Acurácia (0,82), mas isso é enganoso devido ao forte desbalanceamento dos dados (84% permanecem).

✅ Conclusão da Análise

A Regressão Logística é mais indicada para este problema: prioriza a detecção de funcionários em risco (Recall maior), essencial para que o RH possa agir proativamente.

Importância do F1-Score:

  • Acurácia é enganosa em datasets desbalanceados
  • F1-Score equilibra Precisão e Recall para a classe minoritária
  • class_weight='balanced' foi crucial para melhorar a detecção da classe positiva

7 Conclusões e Melhorias

📌 Conclusões

  • Regressão Logística com class_weight='balanced' apresentou o melhor equilíbrio entre detecção de saídas (Recall=0,68) e qualidade das predições (F1=0,48)
  • Fatores mais relevantes para turnover: horas extras, baixa renda, distância de casa, poucos anos de empresa e baixa satisfação com o ambiente de trabalho

🚀 Possíveis Melhorias

  • Testar SMOTE como alternativa ao class_weight='balanced'
  • Explorar outros algoritmos: XGBoost, SVM, Redes Neurais
  • Coletar mais dados para melhorar a generalização
  • Análise de features importantes para ações direcionadas de RH
  • Implementação de sistema de alerta em tempo real
